TECHNICAL ARCHITECTURE
----------------------

1. WebSocket Communication Flow
   - Client connects with JWT token authentication
   - Server validates token and establishes session
   - Client joins exam-specific rooms for targeted communication
   - Real-time events flow bidirectionally
   - Automatic reconnection with exponential backoff
   - Graceful disconnection handling

2. Event System
   - Exam Events: join-exam, leave-exam, submit-answer, submit-exam
   - Notification Events: notification, unread-count, mark-read
   - Admin Events: student-joined, student-left, session-inactive
   - System Events: system-announcement, force-disconnect
   - Activity Events: exam-activity, answer-submitted

3. Data Persistence
   - Notifications stored in PostgreSQL database
   - Session state maintained in memory for performance
   - Activity logs for audit trail and monitoring
   - Proper cleanup and garbage collection

4. Performance Optimizations
   - Memory-efficient session management
   - Automatic cleanup of inactive sessions
   - Efficient event broadcasting with room-based targeting
   - Minimal database queries with proper indexing
   - Connection pooling and resource management
